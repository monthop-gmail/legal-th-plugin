# /triage-nda -- คัดกรอง NDA

> หากเห็น placeholder ที่ไม่คุ้นเคย ดู [CONNECTORS.md](../CONNECTORS.md)

คัดกรอง NDA: @$1

คัดกรอง NDA ขาเข้าตามเกณฑ์มาตรฐาน จัดระดับเพื่อส่งต่อ: อนุมัติมาตรฐาน, ให้ทนายรีวิว, หรือต้องรีวิวเต็มรูปแบบ

**สำคัญ**: คำสั่งนี้ช่วยคัดกรองเบื้องต้น ไม่ใช่คำปรึกษาทางกฎหมาย

## Invocation

```
/triage-nda
```

## Workflow

### ขั้นตอน 1: รับ NDA

รับ NDA ได้ทุกรูปแบบ:
- **อัปโหลดไฟล์**: PDF, DOCX
- **URL**: ลิงก์ไปยัง NDA ในระบบเอกสาร
- **วางข้อความ**: วางข้อความ NDA โดยตรง

### ขั้นตอน 2: โหลด NDA Playbook

ค้นหาเกณฑ์คัดกรอง NDA ใน `legal.local.md`

**หากไม่มี playbook** ใช้ค่าเริ่มต้นตามมาตรฐานไทย:
- ภาระผูกพันร่วมกันทั้งสองฝ่าย (Mutual)
- ระยะเวลา: 2-3 ปี มาตรฐาน, 5 ปี สำหรับความลับทางการค้า
- Standard carveouts: พัฒนาอิสระ, เปิดเผยต่อสาธารณะ, ได้รับจากบุคคลที่สาม, กฎหมายบังคับ
- ไม่มีข้อห้ามชักชวน (non-solicitation) หรือข้อห้ามแข่งขัน (non-compete)
- กฎหมายไทยเป็นกฎหมายที่ใช้บังคับ

### ขั้นตอน 3: คัดกรอง

| เกณฑ์ | ตรวจสอบ |
|-------|--------|
| **Mutual vs Unilateral** | ภาระผูกพันร่วมกันทั้งสองฝ่ายหรือไม่? |
| **นิยามข้อมูลลับ** | ขอบเขตสมเหตุสมผล? ไม่กว้างเกินไป? |
| **ระยะเวลา** | อยู่ในช่วงที่ยอมรับได้? |
| **ข้อยกเว้นมาตรฐาน** | ครบถ้วน? (พัฒนาอิสระ, เปิดเผยสาธารณะ, บุคคลที่สาม, กฎหมายบังคับ) |
| **การเปิดเผยที่อนุญาต** | เปิดเผยต่อพนักงาน ที่ปรึกษา ผู้รับจ้างที่จำเป็นได้? |
| **การคืน/ทำลาย** | ข้อผูกพันเมื่อสิ้นสุดสัญญาสมเหตุสมผล? |
| **Residuals clause** | ถ้ามี กำหนดขอบเขตแคบพอ? |
| **ข้อห้ามชักชวน** | มีข้อห้าม non-solicitation แฝงอยู่? |
| **ข้อห้ามแข่งขัน** | มีข้อห้าม non-compete แฝงอยู่? |
| **คำสั่งคุ้มครองชั่วคราว** | สมเหตุสมผลหรือเอื้อฝ่ายเดียว? |
| **กฎหมายที่ใช้บังคับ** | เขตอำนาจศาลยอมรับได้? |
| **ภาษาของสัญญา** | ไทย/อังกฤษ? กรณีขัดแย้งใช้ภาษาไหน? |
| **ข้อกำหนดผิดปกติ** | มีข้อที่ไม่ควรอยู่ใน NDA? |

### ขั้นตอน 4: จัดระดับ

#### GREEN -- อนุมัติได้
ผ่านทุกเกณฑ์ เป็นมาตรฐานตลาด ไม่มีข้อผิดปกติ
- **ส่งต่อ**: อนุมัติและลงนามตามกระบวนการปกติ

#### YELLOW -- ต้องรีวิว
มีข้อเบี่ยงเบนเล็กน้อยที่ต้องพิจารณา:
- นิยามข้อมูลลับกว้างกว่าปกติแต่ไม่เกินสมเหตุสมผล
- ระยะเวลายาวกว่ามาตรฐานแต่อยู่ในช่วงตลาด
- ขาด carveout บางข้อที่เพิ่มได้
- กฎหมายที่ใช้ไม่ใช่ไทยแต่ยอมรับได้
- **ส่งต่อ**: ให้ทนายรีวิวประเด็นที่พบ

#### RED -- มีปัญหาสำคัญ
มีข้อเบี่ยงเบนที่มีความเสี่ยง:
- ภาระผูกพันฝ่ายเดียวเมื่อต้องการ mutual
- ขาด carveout สำคัญ (เช่น ไม่มีข้อยกเว้นพัฒนาอิสระ)
- มี non-solicitation หรือ non-compete แฝง
- ระยะเวลาไม่สมเหตุสมผล (10+ ปี)
- ข้อกำหนดผิดปกติ (ผูกขาด, สิทธิ์ตรวจสอบ, โอน IP)
- กฎหมายต่างประเทศที่ไม่เป็นมิตร
- **ส่งต่อ**: ต้องรีวิวเต็มรูปแบบ อย่าลงนาม

```
**กฎหมายที่ใช้**: [เขตอำนาจ]
**ภาษา**: [ไทย / อังกฤษ / สองภาษา]

| ภาระผูกพันร่วมกัน | [ผ่าน/ต้องดู/ไม่ผ่าน] | [รายละเอียด] |
| นิยามข้อมูลลับ | [ผ่าน/ต้องดู/ไม่ผ่าน] | [รายละเอียด] |
| [ฯลฯ] | | |

## ประเด็นที่พบ

### [ประเด็น 1 -- YELLOW/RED]
**พบอะไร**: [อธิบาย]
**ความเสี่ยง**: [อะไรอาจเกิดขึ้น]

[ขั้นตอนถัดไป: อนุมัติ / ส่งรีวิวพร้อมหมายเหตุ / ปฏิเสธและเสนอ NDA ขององค์กร]
```

## หมายเหตุ

- หากเอกสารไม่ใช่ NDA จริง (เช่น มีเงื่อนไขทางการค้าแฝง) ให้จัดเป็น RED ทันทีและแนะนำรีวิวสัญญาเต็มรูปแบบ
- NDA ที่เป็นส่วนหนึ่งของสัญญาใหญ่ ให้แจ้งว่าบริบทของสัญญาหลักอาจมีผล
- ตรวจสอบว่า NDA ต้องติดอากรแสตมป์หรือไม่
