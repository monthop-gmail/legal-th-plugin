# /compliance-check -- ตรวจสอบ Compliance กฎหมายไทย

> หากเห็น placeholder ที่ไม่คุ้นเคย ดู [CONNECTORS.md](../CONNECTORS.md)

ตรวจสอบ compliance ของการดำเนินการ ฟีเจอร์ผลิตภัณฑ์ แคมเปญการตลาด หรือโครงการธุรกิจ ตามกฎหมายและระเบียบข้อบังคับของไทย

**สำคัญ**: คำสั่งนี้ช่วยงาน workflow ทางกฎหมาย แต่ไม่ใช่คำปรึกษาทางกฎหมาย ผลการตรวจสอบควรได้รับการยืนยันจากทนายความที่มีคุณสมบัติ

## Usage

```
/compliance-check $ARGUMENTS
```

## สิ่งที่ต้องการจากคุณ

อธิบายสิ่งที่คุณวางแผนจะทำ ตัวอย่าง:
- "เราจะเก็บข้อมูลชีวมิติของพนักงาน"
- "ต้องการส่งข้อมูลลูกค้าไปประมวลผลที่ต่างประเทศ"
- "จะใช้ข้อมูลลูกค้าทำ marketing automation"
- "ต้องการจ้างพนักงานสัญญาจ้างชั่วคราว 200 คน"
- "วางแผนจะเปิดร้านค้าออนไลน์ขายอาหารเสริม"

## Workflow

### ขั้นตอน 1: ทำความเข้าใจโครงการ

ถามรายละเอียดเพิ่มเติมถ้าข้อมูลไม่เพียงพอ:
- **ทำอะไร**: อธิบายกิจกรรม/โครงการ
- **เกี่ยวข้องกับใคร**: ลูกค้า, พนักงาน, คู่ค้า, สาธารณะ
- **ข้อมูลอะไร**: ข้อมูลส่วนบุคคล, ข้อมูลอ่อนไหว, ข้อมูลทางการเงิน
- **ที่ไหน**: ในไทย, ข้ามประเทศ, ออนไลน์

### ขั้นตอน 2: ระบุกฎหมายที่เกี่ยวข้อง

ค้นหากฎหมายที่เกี่ยวข้องผ่าน ~~legal-th:

**กฎหมายหลักที่ต้องพิจารณา:**

| กฎหมาย | เมื่อเกี่ยวข้อง |
|--------|-----------------|
| **PDPA** (พ.ร.บ.คุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562) | เก็บ/ใช้/เปิดเผยข้อมูลส่วนบุคคล |
| **พ.ร.บ.คุ้มครองแรงงาน พ.ศ. 2541** | จ้างงาน สวัสดิการ การเลิกจ้าง |
| **พ.ร.บ.คุ้มครองผู้บริโภค พ.ศ. 2522** | ขายสินค้า/บริการต่อผู้บริโภค |
| **พ.ร.บ.ว่าด้วยธุรกรรมทางอิเล็กทรอนิกส์ พ.ศ. 2544** | ทำธุรกรรมออนไลน์ |
| **พ.ร.บ.ว่าด้วยการกระทำความผิดเกี่ยวกับคอมพิวเตอร์ พ.ศ. 2550** | ระบบคอมพิวเตอร์และข้อมูลจราจร |
| **ประมวลกฎหมายแพ่งและพาณิชย์** | สัญญา นิติกรรม ความรับผิด |
| **ประมวลรัษฎากร** | ภาษี อากรแสตมป์ หัก ณ ที่จ่าย |
| **พ.ร.บ.ลิขสิทธิ์ พ.ศ. 2537** | งานอันมีลิขสิทธิ์ ซอฟต์แวร์ |
| **พ.ร.บ.สิทธิบัตร พ.ศ. 2522** | สิ่งประดิษฐ์ อนุสิทธิบัตร |
| **พ.ร.บ.เครื่องหมายการค้า พ.ศ. 2559** | แบรนด์ เครื่องหมายการค้า |

### ขั้นตอน 3: ตรวจสอบ Compliance

สำหรับแต่ละกฎหมายที่เกี่ยวข้อง ตรวจสอบ:
- ข้อกำหนดที่ต้องปฏิบัติ
- สถานะปัจจุบัน (ปฏิบัติแล้ว / ยังไม่ปฏิบัติ / ไม่ทราบ)
- สิ่งที่ต้องดำเนินการ

### ขั้นตอน 4: ประเมินความเสี่ยง

ประเมินความเสี่ยงของแต่ละประเด็น:
- **สูง**: อาจมีโทษทางอาญา ปรับสูง หรือเสียชื่อเสียงรุนแรง
- **กลาง**: อาจมีค่าปรับ หรือถูกดำเนินคดีแพ่ง
- **ต่ำ**: ความเสี่ยงน้อย แต่ควรปฏิบัติตาม

## Tips

1. **ระบุให้ชัด** — "จะส่ง email ถึงลูกค้าทุกคน" ดีกว่า "จะทำ marketing"
2. **บอกขอบเขตพื้นที่** — กฎหมายต่างกันตามเขตอำนาจ
3. **บอกข้อมูลที่เกี่ยวข้อง** — ข้อมูลส่วนบุคคลอะไรบ้าง เป็นตัวกำหนด compliance ส่วนใหญ่
4. **บอกอุตสาหกรรม** — บางอุตสาหกรรมมีกฎเฉพาะ (การเงิน, สาธารณสุข, อาหาร)
