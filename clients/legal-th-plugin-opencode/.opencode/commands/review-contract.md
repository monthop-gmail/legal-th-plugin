# /review-contract -- รีวิวสัญญาตาม Playbook

> หากเห็น placeholder ที่ไม่คุ้นเคย ดู [CONNECTORS.md](../CONNECTORS.md)

รีวิวสัญญาตาม playbook ขององค์กร วิเคราะห์ทีละข้อ ตรวจข้อเบี่ยงเบน สร้าง redline และวิเคราะห์ผลกระทบทางธุรกิจ

**สำคัญ**: คำสั่งนี้ช่วยงาน workflow ทางกฎหมาย แต่ไม่ใช่คำปรึกษาทางกฎหมาย ผลวิเคราะห์ควรได้รับการตรวจสอบจากทนายความที่มีคุณสมบัติ

## Invocation

```
/review-contract <ไฟล์สัญญาหรือ URL>
```

รีวิวสัญญา: @$1

## Workflow

### ขั้นตอน 1: รับสัญญา

รับสัญญาได้ทุกรูปแบบ:
- **อัปโหลดไฟล์**: PDF, DOCX หรือรูปแบบอื่น
- **URL**: ลิงก์ไปยังสัญญาใน ~~cloud storage หรือระบบเอกสาร
- **วางข้อความ**: วางข้อความสัญญาโดยตรง

หากไม่มีสัญญา ให้ขอจากผู้ใช้

### ขั้นตอน 2: รวบรวมบริบท

ถามผู้ใช้ก่อนเริ่มรีวิว:

1. **คุณอยู่ฝ่ายไหน?** (ผู้ขาย/ผู้ให้บริการ, ผู้ซื้อ/ลูกค้า, ผู้อนุญาต, ผู้รับอนุญาต, คู่ค้า)
2. **กำหนดส่ง**: ต้องเสร็จเมื่อไหร่?
3. **จุดสำคัญ**: มีข้อกังวลเฉพาะไหม? (เช่น "PDPA สำคัญมาก", "ต้องยืดหยุ่นเรื่องระยะเวลา")
4. **บริบทธุรกิจ**: มูลค่าดีล, ความสำคัญเชิงกลยุทธ์, ความสัมพันธ์เดิม

### ขั้นตอน 3: โหลด Playbook

ค้นหา playbook ใน local settings (`legal.local.md`)

Playbook กำหนด:
- **Standard positions**: เงื่อนไขสัญญาที่องค์กรต้องการ
- **Acceptable ranges**: เงื่อนไขที่ยอมรับได้โดยไม่ต้อง escalate
- **Escalation triggers**: เงื่อนไขที่ต้องให้ผู้บังคับบัญชารีวิว

**หากไม่มี playbook:**
- แจ้งผู้ใช้
- เสนอ 2 ทางเลือก:
  1. ช่วยตั้ง playbook (กำหนดจุดยืนสำหรับข้อสำคัญ)
  2. รีวิวตามมาตรฐานกฎหมายไทยทั่วไป
- หากใช้มาตรฐานทั่วไป ระบุชัดเจนว่า "รีวิวตามมาตรฐานกฎหมายไทยทั่วไป"

### ขั้นตอน 4: วิเคราะห์ทีละข้อ

วิเคราะห์สัญญาอย่างเป็นระบบ ครอบคลุม:

| หมวดข้อสัญญา | จุดรีวิวสำคัญ |
|-------------|--------------|
| **ข้อจำกัดความรับผิด** | จำนวนเงินสูงสุด, ฝ่ายเดียว/ร่วมกัน, ค่าเสียหายที่เป็นผลสืบเนื่อง |
| **การชดใช้ค่าเสียหาย** | ขอบเขต, ฝ่ายเดียว/ร่วมกัน, เพดาน, IP, ข้อมูลรั่วไหล |
| **ทรัพย์สินทางปัญญา** | IP เดิม, IP ที่พัฒนาระหว่างสัญญา, work-for-hire, สิทธิ์อนุญาต |
| **คุ้มครองข้อมูลส่วนบุคคล** | DPA/PDPA, ผู้ประมวลผลข้อมูล, แจ้งเหตุละเมิด, โอนข้อมูลข้ามประเทศ |
| **การรักษาความลับ** | ขอบเขต, ระยะเวลา, ข้อยกเว้น, การคืน/ทำลาย |
| **คำรับรองและรับประกัน** | ขอบเขต, ข้อยกเว้น, ระยะเวลา |
| **ระยะเวลาและการเลิกสัญญา** | ระยะเวลา, ต่ออัตโนมัติ, บอกเลิก, ผลของการเลิกสัญญา |
| **กฎหมายที่ใช้บังคับ** | เขตอำนาจศาล, การระงับข้อพิพาท, อนุญาโตตุลาการ |
| **ค่าปรับและเบี้ยปรับ** | เงื่อนไข, จำนวน, ความสมเหตุสมผลตามกฎหมายไทย |
| **ภาษีและอากรแสตมป์** | ภาระภาษี, อากรแสตมป์ (ตามประมวลรัษฎากร) |
| **เหตุสุดวิสัย** | ขอบเขต, การแจ้ง, สิทธิ์เลิกสัญญา |
| **เงื่อนไขการชำระเงิน** | ระยะเวลา, ดอกเบี้ย, ภาษีหัก ณ ที่จ่าย |

### ขั้นตอน 5: จัดระดับข้อเบี่ยงเบน

#### GREEN -- ยอมรับได้
- สอดคล้องกับหรือดีกว่าจุดยืนขององค์กร
- ความแตกต่างเล็กน้อยที่สมเหตุสมผล
- ไม่ต้องดำเนินการ แจ้งเพื่อทราบ

#### YELLOW -- ต้องเจรจา
- นอกจุดยืนมาตรฐาน แต่อยู่ในช่วงที่เจรจาได้
- พบได้ทั่วไปในตลาด แต่ไม่ใช่สิ่งที่องค์กรต้องการ
- **ให้**: redline ที่แก้ไขแล้ว + ท่าทีสำรอง + ผลกระทบทางธุรกิจ

#### RED -- ต้อง Escalate
- นอกช่วงที่ยอมรับได้ หรือมีความเสี่ยงสูง
- ต้องให้ผู้บังคับบัญชาหรือที่ปรึกษาภายนอกรีวิว
- **ให้**: เหตุผลที่เป็น RED + มาตรฐานตลาด + ความเสี่ยง + แนวทาง escalation

### ขั้นตอน 6: สร้าง Redline

สำหรับทุกข้อที่เป็น YELLOW และ RED:
- **ข้อความปัจจุบัน**: อ้างอิงจากสัญญา
- **Redline ที่แนะนำ**: ข้อความทดแทนที่พร้อมใช้
- **เหตุผล**: อธิบายสั้นๆ เหมาะสำหรับส่งให้คู่สัญญา
- **ลำดับความสำคัญ**: ต้องมี / ควรมี / มีก็ดี
- **ท่าทีสำรอง**: หากคู่สัญญาไม่ยอม

### ขั้นตอน 7: สรุปผลกระทบทางธุรกิจ

- **ภาพรวมความเสี่ยง**: ระดับความเสี่ยงโดยรวม
- **3 ประเด็นสำคัญ**: สิ่งที่ต้องแก้ไขเร่งด่วน
- **กลยุทธ์การเจรจา**: แนวทางที่แนะนำ
- **ข้อพิจารณาด้านเวลา**: ปัจจัยเร่งด่วน

## Output Format

```
## สรุปผลรีวิวสัญญา

**เอกสาร**: [ชื่อสัญญา]
**คู่สัญญา**: [ชื่อคู่สัญญาและบทบาท]
**ฝ่ายของคุณ**: [ผู้ขาย/ลูกค้า/อื่นๆ]
**กำหนดส่ง**: [ถ้ามี]
**ฐานการรีวิว**: [Playbook / มาตรฐานกฎหมายไทยทั่วไป]

## ประเด็นสำคัญ

[3-5 ประเด็นหลักพร้อมระดับ GREEN/YELLOW/RED]

## วิเคราะห์ทีละข้อ

### [หมวดข้อสัญญา] -- [GREEN/YELLOW/RED]
**สัญญาระบุว่า**: [สรุปข้อกำหนด]
**จุดยืน Playbook**: [มาตรฐานองค์กร]
**ข้อเบี่ยงเบน**: [อธิบายความต่าง]
**ผลกระทบทางธุรกิจ**: [ความหมายในทางปฏิบัติ]
**Redline ที่แนะนำ**: [ข้อความทดแทน ถ้าเป็น YELLOW/RED]

[ทำซ้ำสำหรับแต่ละข้อสำคัญ]

## กลยุทธ์การเจรจา

[แนวทาง ลำดับความสำคัญ สิ่งที่ยอมได้]

## ขั้นตอนถัดไป

[สิ่งที่ต้องดำเนินการ]
```

## หมายเหตุ

- หากสัญญาเป็นภาษาอังกฤษ ถามว่าต้องการรีวิวเป็นภาษาไทยหรือภาษาเดิม
- สัญญายาวมาก (50+ หน้า) เสนอให้โฟกัสส่วนสำคัญก่อน
- ตรวจสอบว่าสัญญาต้องติดอากรแสตมป์หรือไม่ ตามประมวลรัษฎากร
- ตรวจสอบภาษีหัก ณ ที่จ่ายที่เกี่ยวข้อง
- เตือนเสมอว่าผลวิเคราะห์ต้องได้รับการตรวจสอบจากทนายความ
