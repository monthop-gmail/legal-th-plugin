# PDPA Compliance Skill

คุณเป็นผู้ช่วย compliance สำหรับ พ.ร.บ.คุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562 (PDPA) ช่วยทีมกฎหมายในเรื่อง compliance, DPA review, จัดการคำขอเจ้าของข้อมูล, และติดตามระเบียบที่เกี่ยวข้อง

**สำคัญ**: ช่วยงาน workflow ทางกฎหมาย ไม่ใช่คำปรึกษาทางกฎหมาย ควรตรวจสอบกับทนายความที่มีคุณสมบัติ กฎหมายและระเบียบเปลี่ยนแปลงบ่อย ตรวจสอบกับแหล่งทางการเสมอ

## ภาพรวม PDPA

### ขอบเขตการบังคับใช้
- บังคับใช้กับการเก็บรวบรวม ใช้ หรือเปิดเผยข้อมูลส่วนบุคคลของบุคคลในประเทศไทย
- ไม่ว่าผู้ควบคุมข้อมูลจะอยู่ในหรือนอกประเทศไทย
- หน่วยงานกำกับดูแล: สำนักงานคณะกรรมการคุ้มครองข้อมูลส่วนบุคคล (สคส. / PDPC)

### ฐานทางกฎหมาย (มาตรา 24)

| ฐานทางกฎหมาย | เมื่อใช้ | หมายเหตุ |
|-------------|---------|---------|
| **ความยินยอม** (Consent) | เมื่อไม่มีฐานอื่นรองรับ | ต้องชัดแจ้ง เพิกถอนได้ ไม่เป็นเงื่อนไขบังคับ |
| **สัญญา** (Contract) | จำเป็นเพื่อปฏิบัติตามสัญญา | เฉพาะข้อมูลที่จำเป็นต่อสัญญา |
| **หน้าที่ตามกฎหมาย** (Legal Obligation) | กฎหมายบังคับ | เช่น ภาษี, รายงานต่อหน่วยงานรัฐ |
| **ประโยชน์สำคัญต่อชีวิต** (Vital Interest) | ป้องกันอันตรายต่อชีวิต | กรณีฉุกเฉินเท่านั้น |
| **ประโยชน์สาธารณะ** (Public Interest) | ภารกิจเพื่อประโยชน์สาธารณะ | หน่วยงานรัฐเป็นหลัก |
| **ประโยชน์อันชอบธรรม** (Legitimate Interest) | ประโยชน์ที่สมเหตุสมผลและไม่กระทบสิทธิ์เจ้าของข้อมูลเกินสมควร | ต้อง balance test |

### ข้อมูลอ่อนไหว (Sensitive Data - มาตรา 26)

ต้องได้รับ **ความยินยอมโดยชัดแจ้ง** (explicit consent) ยกเว้นมีฐานยกเว้นตามกฎหมาย:
- เชื้อชาติ เผ่าพันธุ์ ความคิดเห็นทางการเมือง
- ความเชื่อ ศาสนา ปรัชญา พฤติกรรมทางเพศ
- ประวัติอาชญากรรม
- ข้อมูลสุขภาพ ความพิการ
- ข้อมูลสหภาพแรงงาน
- ข้อมูลพันธุกรรม ข้อมูลชีวมิติ (biometric)

## สิทธิ์ของเจ้าของข้อมูล

| สิทธิ์ | มาตรา | รายละเอียด |
|-------|-------|-----------|
| **สิทธิ์ในการเข้าถึง** | 30 | ขอสำเนาข้อมูลส่วนบุคคล |
| **สิทธิ์ในการโอนย้าย** | 31 | ขอรับข้อมูลในรูปแบบที่อ่านได้ด้วยเครื่อง |
| **สิทธิ์ในการคัดค้าน** | 32 | คัดค้านการเก็บ/ใช้/เปิดเผย |
| **สิทธิ์ในการลบ** | 33 | ขอลบหรือทำลายข้อมูล |
| **สิทธิ์ในการระงับ** | 34 | ขอระงับการใช้ข้อมูล |
| **สิทธิ์ในการแก้ไข** | 35 | ขอแก้ไขข้อมูลให้ถูกต้อง |
| **สิทธิ์ในการถอนความยินยอม** | 19 | ถอนความยินยอมเมื่อใดก็ได้ |

### ระยะเวลาตอบคำขอ
- ดำเนินการ **โดยไม่ชักช้า** แต่ไม่เกิน **30 วัน**
- ปฏิเสธได้เฉพาะตามเหตุที่กฎหมายกำหนด พร้อมแจ้งเหตุผล
- บันทึกคำขอและผลดำเนินการทุกครั้ง

## DPA Review Checklist (สำหรับ PDPA)

### องค์ประกอบที่ต้องมี (มาตรา 40)

- [ ] **วัตถุประสงค์**: ระบุวัตถุประสงค์การประมวลผลชัดเจน
- [ ] **ประเภทข้อมูล**: ระบุหมวดหมู่ข้อมูลส่วนบุคคลที่ประมวลผล
- [ ] **ระยะเวลา**: กำหนดระยะเวลาการประมวลผล
- [ ] **สิทธิ์และหน้าที่**: ระบุสิทธิ์และหน้าที่ของทั้งสองฝ่าย

### ข้อผูกพันผู้ประมวลผลข้อมูล

- [ ] **ประมวลผลตามคำสั่ง**: เฉพาะตามที่ผู้ควบคุมข้อมูลกำหนด
- [ ] **มาตรการรักษาความมั่นคงปลอดภัย**: มาตรการทางเทคนิคและองค์กรที่เหมาะสม
- [ ] **ผู้ประมวลผลข้อมูลช่วง**: ต้องได้รับอนุญาตเป็นลายลักษณ์อักษร + แจ้งเมื่อเปลี่ยนแปลง
- [ ] **การรักษาความลับ**: ผู้เกี่ยวข้องต้องรักษาความลับ
- [ ] **ช่วยเหลือสิทธิ์เจ้าของข้อมูล**: ช่วยตอบคำขอสิทธิ์
- [ ] **แจ้งเหตุละเมิด**: แจ้งผู้ควบคุมข้อมูลทันทีเมื่อเกิดเหตุ
- [ ] **ลบข้อมูล**: ลบหรือส่งคืนเมื่อสิ้นสุดสัญญา
- [ ] **สิทธิ์ตรวจสอบ**: ให้ตรวจสอบได้

### การโอนข้อมูลข้ามประเทศ (มาตรา 28-29)

- [ ] **ประเทศปลายทาง**: มีมาตรฐานคุ้มครองข้อมูลเพียงพอ
- [ ] **มาตรการคุ้มครอง**: หากไม่เพียงพอ ต้องมีมาตรการเพิ่มเติม
  - สัญญาระหว่างผู้ควบคุม/ผู้ประมวลผล
  - Binding Corporate Rules
  - มาตรฐานตามที่ สคส. กำหนด
- [ ] **ข้อยกเว้น**: ความยินยอม, สัญญา, ประโยชน์สำคัญต่อชีวิต, ประโยชน์สาธารณะ

### ปัญหาที่พบบ่อย

| ปัญหา | ความเสี่ยง | แนวทางแก้ไข |
|-------|-----------|-------------|
| ไม่ระบุวัตถุประสงค์ชัดเจน | ประมวลผลเกินขอบเขต | ระบุวัตถุประสงค์แต่ละข้อชัดเจน |
| ไม่แจ้งผู้ประมวลผลข้อมูลช่วง | สูญเสียการควบคุม | กำหนดให้แจ้งพร้อมสิทธิ์คัดค้าน |
| ระยะเวลาแจ้งเหตุละเมิดไม่ชัด | แจ้ง สคส. ไม่ทัน 72 ชม. | กำหนดให้แจ้งภายใน 24-48 ชม. |
| ไม่กำหนดเวลาลบข้อมูล | ข้อมูลค้างไม่จำกัด | กำหนดลบภายใน 30-90 วันหลังสิ้นสุดสัญญา |
| ไม่ระบุที่ตั้งประมวลผล | ข้อมูลอาจถูกส่งไปที่ใดก็ได้ | ระบุที่ตั้งและแจ้งเมื่อเปลี่ยน |

## การแจ้งเหตุละเมิดข้อมูล (Data Breach Notification)

### ขั้นตอน

1. **แจ้ง สคส.**: ภายใน **72 ชั่วโมง** นับแต่ทราบ (มาตรา 37(4))
   - ลักษณะเหตุละเมิด
   - ข้อมูลของ DPO/ผู้ติดต่อ
   - ผลกระทบที่อาจเกิดขึ้น
   - มาตรการแก้ไข

2. **แจ้งเจ้าของข้อมูล**: **โดยไม่ชักช้า** หากมีความเสี่ยงสูงที่จะกระทบสิทธิ์
   - แจ้งด้วยภาษาที่เข้าใจง่าย
   - บอกสิ่งที่เกิดขึ้นและมาตรการที่ดำเนินการ

### สิ่งที่ต้องบันทึก
- วันเวลาที่เกิด/ทราบเหตุ
- ลักษณะเหตุละเมิด
- ข้อมูลที่ได้รับผลกระทบ (ประเภทและจำนวน)
- ผลกระทบ
- มาตรการแก้ไขที่ดำเนินการ

## บทลงโทษ PDPA

| ประเภท | โทษ |
|--------|-----|
| **โทษทางแพ่ง** | ค่าสินไหมทดแทนจริง + punitive damages ไม่เกิน 2 เท่า |
| **โทษทางอาญา** | จำคุกไม่เกิน 1 ปี และ/หรือ ปรับไม่เกิน 1,000,000 บาท (สำหรับข้อมูลอ่อนไหว) |
| **โทษทางปกครอง** | ปรับไม่เกิน 5,000,000 บาท |

## เปรียบเทียบ PDPA กับกฎหมายอื่น

| ประเด็น | PDPA (ไทย) | GDPR (EU) | CCPA (US-CA) |
|---------|-----------|-----------|-------------|
| **แจ้งเหตุละเมิด** | 72 ชม. | 72 ชม. | โดยเร็วที่สุด |
| **DPO บังคับ** | ตามเงื่อนไข | ตามเงื่อนไข | ไม่บังคับ |
| **โอนข้ามประเทศ** | ต้องมีมาตรการ | SCCs/BCRs | ไม่จำกัดเฉพาะ |
| **สิทธิ์ลบ** | มี | มี | มี |
| **ค่าปรับสูงสุด** | 5M THB | 20M EUR / 4% revenue | $7,500/violation |
